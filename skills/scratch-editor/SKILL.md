# Scratch Editor

Edit and modify existing Scratch 3.0 projects — add features, fix bugs, change
sprites or logic, and regenerate updated .sb3 files.

## When to Use This Skill

Use this skill when the user wants to:
- **Modify** an existing project ("add a score display", "make the ball faster")
- **Fix a bug** in a project ("the paddle doesn't move", "game doesn't restart")
- **Extend** a project ("add a second level", "add sound effects")
- **Inspect** what's in an existing project before editing

### Trigger Examples

- "edit the pong game to add sound"
- "modify my platformer to add lives"
- "load my catch game and make the objects fall faster"
- "what sprites are in my project?"
- "update the ball speed in pong_game.sb3"

## Workflow

### Step 1: Discover Available Projects

Use `list_projects()` to show the user what's in the output directory:

```
list_projects()
```

### Step 2: Load the Project

**Prefer spec-based editing** — if a `_spec.json` exists, load it:

```
load_spec("pong_game_spec.json")
```

If only an `.sb3` exists, inspect it first for orientation:

```
inspect_sb3("pong_game.sb3")
```

Then load the raw project JSON if deeper editing is needed:

```
load_sb3_project("pong_game.sb3")
```

### Step 3: Plan Changes

Before editing, summarize what will change:
1. Which sprites are affected
2. Which blocks will be added/modified
3. Any new variables or broadcasts needed

### Step 4: Apply Changes

**Spec-based flow (preferred):**
1. Modify the spec JSON in memory based on the user's request
2. Save the modified spec: `save_spec("pong_game_spec.json", <modified_json>)`
3. Regenerate the .sb3 using run_bash:
   ```
   python skills/scratch-coder/scripts/generate_sb3.py output/pong_game_spec.json --output output/pong_game.sb3 --overwrite
   ```

**Raw .sb3 flow (fallback when no spec exists):**
1. Load full project.json via `load_sb3_project()`
2. Modify the project JSON structure directly
3. Save modified project.json as a new spec: `save_spec("project_edited_spec.json", <modified_json>)`
4. Regenerate using generate_sb3.py

### Step 5: Verify and Report

- Confirm the .sb3 was regenerated successfully
- Tell the user what changed and where the file is

## Editing Guidelines

### Adding a New Sprite

Add an entry to the `"sprites"` array in the spec:

```json
{
  "name": "NewSprite",
  "costume": "cat",
  "x": 0,
  "y": 0,
  "blocks": {
    "BLOCK_A": {
      "opcode": "event_whenflagclicked",
      "next": null,
      "parent": null,
      "inputs": {},
      "fields": {},
      "shadow": false,
      "topLevel": true,
      "x": 0,
      "y": 0
    }
  }
}
```

### Adding a Variable

Add to `stage.variables`:

```json
"stage": {
  "variables": {
    "VAR_ID_speed": ["Speed", 5]
  }
}
```

### Modifying Block Logic

Blocks reference each other by ID in `"next"` and `"parent"` fields.
To insert a block between BLOCK_A and BLOCK_B:
1. Create a new block ID (e.g. `"BLOCK_NEW"`)
2. Set `BLOCK_A.next = "BLOCK_NEW"`
3. Set `BLOCK_NEW.next = "BLOCK_B"`, `BLOCK_NEW.parent = "BLOCK_A"`
4. Set `BLOCK_B.parent = "BLOCK_NEW"`

### Common Changes

| Goal | What to modify |
|------|---------------|
| Faster movement | Increase `STEPS` input value in `motion_movesteps` |
| Add scoring | Add variable to stage, add `data_changevariableby` block |
| Add sound | Add sound entry to sprite, add `sound_play` block |
| More lives | Add `Lives` variable, decrement on collision |
| New level | Add backdrop, broadcast `"next_level"`, switch backdrop on receive |
